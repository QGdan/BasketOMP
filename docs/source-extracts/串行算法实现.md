### common.h - 公共数据结构模块

文件位置：include/common.h

功能说明：定义整个项目使用的公共数据结构和类型别名，为其他所有模块提供统一的数据类型基础。

主要数据结构：

| 数据结构 | 类型别名 | 说明 |
| --- | --- | --- |
| Basket | vector<int> | 单个订单的商品ID列表 |
| CooccurrenceMap | unordered_map<string, int> | 共现矩阵，key="a,b"，value=共现次数 |
| UserHistory | unordered_map<int, vector<int>> | 用户购买历史，user_id → 商品列表 |
| PopularityMap | unordered_map<int, int> | 商品流行度，product_id → 出现次数 |
| RecommendationResult | unordered_map<int, vector<int>> | 推荐结果，user_id → Top-K商品列表 |
| EvaluationMetrics | struct | 评估指标：Hit Rate、Recall 等 |

### cooccurrence - 共现统计模块

文件位置：src/cooccurrence.h 和 src/cooccurrence.cpp

功能说明：核心算法模块，负责统计所有商品对在同一订单中的共现次数，构建商品共现图。

算法原理：

对于每个订单，将订单中所有商品两两组合，形成共现对：

text

订单: [p1, p2, p3, p4]共现对: (p1,p2), (p1,p3), (p1,p4), (p2,p3), (p2,p4), (p3,p4)

如果商品对 (a,b) 在多个订单中出现，共现权重累加。

主要函数：

| 函数名 | 功能说明 | 输入 | 输出 |
| --- | --- | --- | --- |
| computeCooccurrence() | 串行统计所有商品对的共现次数 | 订单列表 | 共现矩阵 |
| truncateNeighbors() | 截断每个商品的邻居，只保留 Top-N | 共现矩阵、N值 | 截断后的邻居图 |

时间复杂度：O(∑ n_i²)，其中 n_i 为第 i 个订单的商品数量。

空间复杂度：O(P × N)，P 为商品数，N 为 Top-N 邻居数。

### recommendation - 推荐生成模块

文件位置：src/recommendation.h 和 src/recommendation.cpp

功能说明：为每个用户生成个性化的 Top-K 商品推荐列表。

评分函数：

| 参数 | 含义 | 默认值 |
| --- | --- | --- |
| fuser(u,p)fuser​(u,p) | 用户 u 历史上购买商品 p 的次数 | - |
| fcooc(u,p)fcooc​(u,p) | 商品 p 与用户历史商品的共现强度 | - |

| αα | 用户历史频率权重 | 1.0 |

### main.cpp - 主程序模块

文件位置：src/main.cpp

功能说明：整合所有模块，完成完整的数据加载→共现统计→推荐生成→评估的流程。

命令行参数：

bash

./basket_recommender <数据目录> <K值> <Top-N邻居数>

| 参数 | 说明 | 示例 |
| --- | --- | --- |
| 数据目录 | 包含 CSV 文件的目录路径 | data/small/ |
| K值 | 推荐列表长度 | 10 |
| Top-N邻居 | 每个商品保留的邻居数 | 50 |

使用说明

1 编译

bash

mkdir build && cd build

cmake ..

make

2 运行

bash

# 使用 small 数据集，K=10，Top-50 邻居

./basket_recommender ../data/small/ 10 50

# 使用 medium 数据集，K=20

./basket_recommender ../data/medium/ 20 50

# 使用 large 数据集

./basket_recommender ../data/large/ 10 100

3 输出说明

程序输出包含以下部分：

数据集统计：订单数、用户数、商品数

各阶段运行时间：加载、共现统计、推荐生成、评估

推荐样例：前 5 个用户的 Top-K 推荐

评估结果：Hit Rate@K 和 Recall@K
